0Инструкция для Claude Code

  • Этот файл — канон. Палитра, типографика, пути силуэта, якоря компонентов и спецификации берутся из design-tokens JSON в шапке. Не вводи новые цвета и шрифты.
  • Копируй эталоны. Каждый компонент ниже — рабочая разметка с классами (.u-btn, .u-chip, .u-card, .heatbar, .drl, .gantt…). Переноси CSS-блоки как есть.
  • Один файл, офлайн. Итоговое приложение — самодостаточный HTML: vanilla JS, inline SVG, без CDN, без внешних шрифтов, без Leaflet. Скелет и тяжёлые ассеты — base64.
  • Hero — сэндвич. Растровый скелет подложкой + absolute SVG-оверлей в координатах кадра 1672×941. Якоря компонентов — в heroOverlay.anchorsPx.
  • Машина — только из силуэта. Любое изображение машины в UI (карточка, узел, строка, тултип) строится из silhouette.pathSimple. Никаких эмодзи, иконок «车» и чужих клипартов.
  • Семь тактов — в tacts. Порядок, названия и флаг «батарея снята» — оттуда; это же источник для чек-листов машин.
  • Состояние — в одном объекте, рендер — чистые функции от состояния. localStorage для сохранения. Числа — tabular-nums, mono.

1Концепция

Тёмная студия со съёмки UMO 5 — база интерфейса. Семантика тепла — градиент задней световой полосы автомобиля. Смысл: холодный электромобиль получает тёплый автономный отопитель → холодный монохромный UI, где теплом заполняется только прогресс установки. Красный Яндекса — энергия действия и хаб нарезки бамперов. Никакой радуги: три температуры — монохром, тепло, холод — и одна кнопка-красный.

2Палитра

Тепловой градиент linear-gradient(90deg,#FF2E1A,#FF8A00,#FFC93D) — только заполнение прогресса. Done-состояние — приглушённый #B8862B, активное — #FF8A00, завершённое целиком — #FFC93D.

4Сигнатурные мотивы

Тепловая полоса кампании

Оснащено 128 из 41231%

Из задней световой полосы UMO. Глобальный статус — всегда вверху экрана. Один градиент на весь интерфейс.

DRL-полосы загрузки

Из передних фар. Слот = машина в день: заполненный / контурный. Пробег света (класс sweep) — только в момент завершения машины.

Гранёный узел (диск колеса)

Город — заливка, обводка теплеет с долей оснащения. Сервис — холодный пунктирный контур. Хаб — красный, пульсирует при переполнении буфера.

Skew-бейджи и чипы

действие в работе нейтральный

Контент внутри контр-наклоняется на +12°, чтобы текст стоял ровно (вложенный span).

Мини-карточка машины — атомарный юнит игры

Один компонент — три контекста: список парка, строка расписания, тултип узла карты. 7 точек прогресса = 7 тактов из tacts.

7Hero-анатомия · сэндвич подложка + оверлей

Батарея опускается на тактах Т3–Т6 — магистраль, бак, труба и выхлоп физически недоступны без её снятия. Это главный драйвер нормо-часов, интерфейс обязан это показывать.

8Карта · схема городов

Карта РФ — собственная SVG-схема (узлы по псевдо-геокоординатам), не тайловая. Тёплый пунктир — машины к сервису, красный — бампера в хаб и обратно. Размер узла ∝ парку/мощности; обводка города теплеет от cold к warm по доле оснащения.

Бампер — самостоятельный объект со своим жизненным циклом, отвязанным от машины. Метрики потока: буфер хаба, время цикла, бампера в пути. Машина не может закрыть Т7 без вернувшегося бампера — жёсткая зависимость в модели.

11Правила и запреты

  • Один тёплый смысл. Градиент и янтарь — только прогресс установки Бинара, топливо, тепло. Появление тепла в третьем значении = поломка системы.
  • Прогресс = заполнение, не смена цвета. Статус кодируется долей заполнения тепловой шкалы, а не переключением зелёный/красный.
  • Красный — дефицитный ресурс. Один primary-баттон на экран, хаб, критический алерт. Всё.
  • Запрещено: светофорные статусы, радуга категорий, синие/зелёные акценты, эмодзи в UI, чужие иконки машин, тени-свечения, parallax, 3D, частицы.
  • Запрещено: внешние зависимости — CDN, веб-шрифты, тайловые карты. Файл открывается офлайн двойным кликом.
  • Доступность: prefers-reduced-motion гасит все анимации; текст ≥ 11px; контраст текста к фону ≥ 4.5:1 (проверено для ts/tp на bg/panel).
  • Числа: mono + tabular-nums везде, где значения меняются. Округление — целые для счётчиков, 0–1 знак для процентов.
